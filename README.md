## 24.04.30
오브젝트 풀링 사용해야 하는 이유
유니티 게임 오브젝트는 오브젝트가 생성 후 파괴될때 GC에 의해 메모리 누수가 발생된다.
자주 파괴되고 자주 생성되는 게임 오브젝트는 오브젝트 풀링 방식으로 메모리 누수를 방지할 수 있다.

오브젝트가 파괴될때 리소스 누수가 발생한다. 그래서 게임이 종료될때, 씬이 변경될때 1번만 진행하게 한다
방식은 생성한 오브젝트들을 컬렉션에 보관하고, 필요할 때 마다 생성한 오브젝트를 활성화 필요없느 ㄴ오브젝트는 비활성화 하는 방시을 사용한다.

정적 오브젝트 풀링, 동적 오브젝트 풀링
정적 : 한번 오브젝트 풀링 할때 데이터를 생성한 후 그 수가 고정된 것
동적 : 최초에 오브젝트 풀링 후 활성화할 데이터가 부족하면 그 때마다 새로운 오브젝트를 추가해주는 방식
